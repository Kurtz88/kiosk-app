# Windows에서 키오스크 자동 실행

`bat\start_kiosk.bat`을 부팅·로그인 후 자동으로 돌리는 대표 방법입니다.

## 방법 1: 시작 프로그램 (가장 간단)

1. 프로젝트 폴더에서 **`bat\install-windows-autostart.bat`** 을 한 번 실행합니다.
2. 다음 로그인부터 `bat\start_kiosk.bat`이 자동 실행됩니다.

**해제:** `Win` + `R` → `shell:startup` 입력 → `RestaurantKiosk` 바로가기 삭제.

## 방법 2: 작업 스케줄러 (더 안정적으로 쓰고 싶을 때)

1. `Win` 키 → **작업 스케줄러** 실행.
2. **기본 작업 만들기**  
   - 트리거: **로그온할 때** (또는 **시작할 때** — 로그인 없이 띄우려면 고급 설정 필요)  
   - 동작: **프로그램 시작**  
   - 프로그램: `cmd.exe`  
   - 인수 추가: `/c "경로\bat\start_kiosk.bat" nopause` (자동 실행 시 `pause` 에서 멈추지 않게)  
   - **시작 위치:** 프로젝트 폴더 전체 경로 (예: `D:\kiosk-restaurant-app`)
3. (선택) **지연** 30초 — 네트워크·디스크 준비 후 실행.

## 방법 3: 레지스트리 실행 항목

`Win` + `R` → `regedit`

- 사용자만:  
  `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`  
  문자열 값 추가 → 이름 예: `Kiosk`, 값: `"D:\경로\bat\start_kiosk.bat"` (경로에 공백 있으면 따옴표)

## 키오스크 PC에서 같이 쓰는 설정

- **자동 로그인:** 설정 → 계정 → 로그인 옵션 (또는 `netplwiz`에서 사용자 암호 생략).
- **전원:** 절전·화면 끄기 비활성화.
- **업데이트 후 재부팅:** 업무 시간 외에만 적용되게 조정.

`bat\start_kiosk.bat`은 먼저 프로젝트 루트(`%~dp0..`)로 이동한 뒤, Node 서버는 **제목이 `Kiosk Server` 인 별도 CMD 창**에서 돌게 합니다(런처만 닫아도 서버는 유지). 수동 실행 시 마지막에 `pause`가 있어 안내를 읽을 수 있고, 자동 실행 시에는 **`start_kiosk.bat nopause`** 로 `pause`를 건너뜁니다. 바로가기·작업 스케줄러는 **시작 위치(작업 폴더)를 프로젝트 루트**로 맞추면 됩니다.

## 자동 실행이 안 될 때 (점검 순서)

1. **등록을 했는지**  
   `bat\install-windows-autostart.bat` 을 **한 번 실행**해야 `시작 프로그램`에 `RestaurantKiosk` 바로가기가 생깁니다. 실행하지 않았다면 자동으로 돌지 않습니다.

2. **“부팅 직후”가 아니라 “로그인 직후”**  
   시작 프로그램 항목은 **Windows에 사용자가 로그인한 뒤**에만 실행됩니다. 전원만 켜고 로그인 화면에서 멈춰 있으면 실행되지 않습니다. 키오스크 PC는 **자동 로그인**을 켜 두는 경우가 많습니다. (설정 → 계정 → 로그인 옵션, 또는 `netplwiz`)

3. **다른 계정으로 로그인**  
   바로가기는 **등록한 그 Windows 사용자**의 시작 폴더에만 들어갑니다. 다른 사용자로 로그인하면 해당 계정에서는 실행되지 않습니다.

4. **바로가기가 있는지**  
   `Win` + `R` → `shell:startup` 입력 → `RestaurantKiosk` 바로가기가 보이는지 확인합니다. 없으면 `install-windows-autostart.bat` 을 다시 실행합니다.

5. **시작 프로그램에서 꺼짐**  
   `Ctrl` + `Shift` + `Esc` → **시작 프로그램** 탭에서 `RestaurantKiosk` 가 **사용 안 함**으로 되어 있지 않은지 확인합니다.

6. **프로젝트 폴더를 옮긴 뒤**  
   폴더 경로가 바뀌면 바로가기가 깨질 수 있습니다. 그때도 `install-windows-autostart.bat` 을 다시 실행해 등록을 새로 하세요.

7. **USB/네트워크 드라이브**  
   프로젝트가 USB나 네트워크 드라이브에만 있고, 로그인 직후에는 아직 연결되지 않으면 `npm` / `node` 가 실패할 수 있습니다. 가능하면 **로컬 디스크**에 두는 것이 안전합니다.
